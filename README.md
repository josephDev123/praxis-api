# Praxis Media API

Task management API built with TypeScript, Express, Prisma, and PostgreSQL.

The app currently provides:

- a health check endpoint
- user registration and login
- authenticated task creation, update, listing, and deletion

Authentication is cookie-based. After a successful login, the server sets a `token` cookie, and all task endpoints require that cookie.

## Tech Stack

- Node.js
- TypeScript
- Express 5
- Prisma ORM
- PostgreSQL
- Zod
- Pino

## Prerequisites

Assume a fresh machine.

Install the following first:

- Node.js 20 or newer
- npm
- PostgreSQL 17 or newer

You also need a PostgreSQL database that your local machine can connect to.

## Environment Variables

Create a local env file from the example:

```bash
cp .env.example .env
```

On Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

Required variables:

- `PORT`: HTTP server port. Defaults to `4000` if omitted.
- `LOG_LEVEL`: One of `fatal`, `error`, `warn`, `info`, `debug`, `trace`.
- `LOG_PRETTY`: `true` or `false`.
- `DATABASE_URL`: PostgreSQL connection string used by Prisma.
- `JWT_SECRET`: Secret used to sign and verify auth tokens.
- `OPENAI_API_KEY`: Required by the current config loader even though no active endpoint uses it yet.

Example PostgreSQL URL format:

```text
postgresql://USER:PASSWORD@localhost:5432/DATABASE_NAME
```

## Local Setup

1. Install dependencies:


2. Create `.env` from `.env.example` and fill in your real values.

## API Overview

Base URL:

```text
http://localhost:4000
```

All routes are mounted under `/api`.

### Health

#### `GET /api/health`

Checks whether the service is up.

Response:

```json
{
  "message": "healthy"
}
```

### Auth

Auth routes are rate-limited.

#### `POST /api/auth/register`

Creates a new user account.

Request body:

```json
{
  "email": "jane@example.com",
  "password": "Secure@123",
  "name": "Jane Doe"
}
```

Validation rules:

- `email` must be a valid email address
- `password` must be at least 6 characters
- `password` must contain at least one uppercase letter
- `password` must contain at least one special character
- `name` is required and limited to 100 characters

Success response:

```json
{
  "message": "User register successfully",
  "data": {
    "id": 1,
    "email": "jane@example.com",
    "name": "Jane Doe"
  }
}
```

#### `POST /api/auth/login`

Authenticates a user and sets a `token` cookie.

Request body:

```json
{
  "email": "jane@example.com",
  "password": "Secure@123"
}
```

Success response:

```json
{
  "message": "login successful",
  "data": {
    "id": 1,
    "email": "jane@example.com",
    "name": "Jane Doe"
  }
}
```

Cookie behavior:

- cookie name: `token`
- `httpOnly: true`
- `secure: true`
- `sameSite: "none"`
- max age: 1 hour

Important local testing note: because the cookie is marked `secure`, browsers usually will not send it over plain `http://localhost` in normal cross-site usage. If you test from a browser client, expect to use HTTPS or adjust the server configuration in code. Tools like Postman can still be used if they preserve cookies manually.

### Tasks

All task routes require the `token` cookie set by login.

Task routes are also rate-limited.

#### `POST /api/task/create`

Creates a task for the authenticated user.

Request body:

```json
{
  "title": "Ship README",
  "description": "Write project documentation",
  "dueDate": "2026-04-10T12:00:00.000Z",
  "priority": "HIGH",
  "status": "TODO"
}
```

Required fields:

- `title`
- `description`
- `dueDate`

Optional fields:

- `priority`: `LOW`, `MEDIUM`, `HIGH`
- `status`: `TODO`, `IN_PROGRESS`, `DONE`

Success response:

```json
{
  "msg": "Task created successfully",
  "data": {
    "id": 1,
    "title": "Ship README",
    "description": "Write project documentation",
    "dueDate": "2026-04-10T12:00:00.000Z",
    "priority": "HIGH",
    "status": "TODO",
    "userId": 1,
    "createdAt": "2026-04-03T00:00:00.000Z",
    "updatedAt": "2026-04-03T00:00:00.000Z"
  }
}
```

#### `PATCH /api/task/update/:id`

Updates a task owned by the authenticated user.

Path params:

- `id`: numeric task id

Body fields are optional. You can send any subset of:

```json
{
  "title": "Updated title",
  "description": "Updated description",
  "dueDate": "2026-04-12T12:00:00.000Z",
  "priority": "MEDIUM",
  "status": "IN_PROGRESS"
}
```

Success response:

```json
{
  "msg": "Task updated successfully",
  "data": {
    "id": 1
  }
}
```

Note: the exact returned `data` shape depends on the Prisma update result and will include the updated task record.

#### `GET /api/task/tasks`

Lists tasks for the authenticated user.

Query params:

- `page`: defaults to `1`
- `limit`: defaults to `10`
- `status`: optional filter, uppercased before query
- `priority`: optional filter, uppercased before query

Example:

```text
GET /api/task/tasks?page=1&limit=10&status=TODO&priority=HIGH
```

Success response:

```json
{
  "msg": "Tasks retrieved successfully",
  "data": [
    {
      "id": 1,
      "title": "Ship README",
      "description": "Write project documentation",
      "dueDate": "2026-04-10T12:00:00.000Z",
      "priority": "HIGH",
      "status": "TODO",
      "userId": 1,
      "createdAt": "2026-04-03T00:00:00.000Z",
      "updatedAt": "2026-04-03T00:00:00.000Z"
    }
  ]
}
```

#### `DELETE /api/task/delete/:id`

Deletes a task owned by the authenticated user.

Path params:

- `id`: numeric task id

Success response:

```json
{
  "msg": "Task deleted successfully",
  "data": {
    "id": 1
  }
}
```

Note: the exact returned `data` shape depends on the Prisma delete result and will include the deleted task record.

## Error Behavior

Errors are returned as JSON from the global error middleware.

Typical patterns:

- `400 Bad Request`
  - missing required fields
  - schema validation errors
  - duplicate registration email
  - invalid login credentials
- `401 Unauthorized`
  - missing auth cookie
- `403 Forbidden`
  - invalid or expired token
  - trying to update/delete another user's task
- `500 Internal Server Error`
  - unexpected server-side failures

Example error response:

```json
{
  "name": "ValidateError",
  "message": "email:Invalid email address"
}
```
